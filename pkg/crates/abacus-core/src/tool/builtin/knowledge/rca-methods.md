# RCA 方法目录 — 20 种系统化根因分析方法

按场景选择 1-3 种方法深入，不要方法跳转。

## 场景 → 方法速查

| 场景 | 最佳方法 |
|------|---------|
| 未知原因，多变量 | 1 分解法、8 单变量法 |
| 回归（曾经能用） | 3 回退法、2 对比法 |
| 间歇性失败 | 10 复现法、16 静候法 |
| 错误信息指向某处 | 5 逆推法、13 依赖链追溯 |
| 复杂系统，多层 | 17 分层剥离法、11 排除法 |
| 数据看起来不对 | 7 透视法、9 边界法 |
| 需要理解未知代码 | 14 日志注入法、15 时间回溯法 |
| 找不到模式 | 18 离群分析、4 假设法 |

## 方法详解

### 1. 分解法 (Divide & Conquer)
对半分问题空间，测试哪半包含 bug，递归缩小。
*例：页面慢 → 分 DNS/前端/服务器 → 服务器 90% → 分 DB/渲染/中间件 → DB 瓶颈*

### 2. 对比法 (Comparison)
对比 working vs failing case，找差异。
*例：Chrome 正常 IE 报错 → 对比请求/响应/HTML → IE 不支持 ES6*

### 3. 回退法 (Rollback)
回滚到已知好状态，逐个重新应用变更。
*例：部署后登录失败 → 回滚稳定版 → 逐个 apply commit → JWT 过期配置未生效*

### 4. 假设法 (Hypothesis Testing)
"如果 X 则 Y 应该 Z"，预测→测试→确认/排除。
*例：API 偶尔 500 → 假设"连接池泄漏" → 重启应恢复正常 ✓*

### 5. 逆推法 (Reverse Inference)
从失败点反向追溯因果链。
*例：订单未发邮件 → MQ 无任务 → 日志"SMTP 连接拒绝" → 防火墙变更*

### 6. 尝试法 (Trial & Error)
搜索空间小+每次快试，快速迭代。
*例：build 符号找不到 → clean rebuild → 更新依赖 → 检查大小写 → 索引缓存问题*

### 7. 透视法 (Look Inside)
不信表面，检查内部状态。
*例：API 返回空数组但数据存在 → SQL 日志/ORM 映射/序列化 → null 字段导致 Jackson 过滤整条记录*

### 8. 单变量法 (Single Variable)
一次只改一个因素，隔离变量。
*例：负载测试 TPS 波动 → 固定并发/数据/硬件调 JVM → 2GB 频繁 GC，4GB 稳定*

### 9. 边界法 (Boundary Testing)
测试空/null/零/最大/最小/溢出。
*例：金额计算错误 → 测 0/0.01/最大值/负数 → 除数为零缺检查*

### 10. 复现法 (Reproduction)
找最小可靠复现步骤。
*例：并发 bug 偶发 → 固定线程数/数据/跑 1000 次 Thread.sleep(1) → 稳定死锁复现*

### 11. 排除法 (Elimination)
逐个禁用/移除组件，定位相关项。
*例：Spring Boot 启动失败 → 逐个注释 @Component → 特定 DataSource 配置冲突*

### 12. 置换法 (Substitution)
用已知好的组件替换可疑组件。
*例：JAR 跑不起来 → 换 JRE（同样问题）→ 用已知好的 JAR 跑这个 JRE（正常）→ 原 JAR 损坏*

### 13. 依赖链追溯 (Chain Tracing)
沿依赖链走，bug 常不在症状处。
*例：CORS 失败 → 检查请求 Origin → 后端响应头 → 网关层剥了 header → 配置错误*

### 14. 日志注入法 (Log Injection)
在决策点加日志，看实际执行路径。
*例：多线程丢记录 → 每步加"进/出+ID" → finally 块提前清队列*

### 15. 时间回溯法 (Time Travel)
从失败时间点向前追溯：什么变了？
*例：配置被覆盖 → etcd 变更历史 → 10:23 批量更新 → CI 脚本没检查环境*

### 16. 静候法 (Wait & Observe)
间歇性问题，延长观察。
*例：服务一周后 OOM → 监控 GC/heap dump → 缓存线性增长无过期*

### 17. 分层剥离法 (Layer Stripping)
绕过外层直接测核心，逐层加回直到失败。
*例：A 调 B 超时 → 从 A 直接 curl B 的 IP:端口（正常）→ sidecar 限流配置太低*

### 18. 离群分析 (Outlier Analysis)
失败案例有什么共同点？
*例：部分用户登不上 → 检查 token → token 含换行符 → base64 解码失败*

### 19. 强制失败法 (Force Failure)
主动诱导失败条件，验证理解。
*例：测重试逻辑 → mock 前两次返回 500，第三次 200 → 验证客户端确实重试*

### 20. 橡皮鸭法 (Rubber Ducking)
向假想同事解释问题，结构化过程暴露答案。
*例：解释为什么条件总走 false → 说着说着发现用了 `=` 而不是 `==`*
